\section{Introduction}
The longest increasing subsequence (LIS for short) problem is a classic algorithmic problem with applications in areas such as sequence analysis, pattern discovery, and optimization. Given a sequence of numbers, the task is to find a longest subsequence whose elements appear in strictly increasing order while preserving their original order, or to compute length of such sequence.
Although the LIS problem has been studied extensively in many variants, the trade-off between time and space remains relevant when the input is large or memory is limited. In this paper, we focus on computing both the length of an LIS and an actual LIS using small auxiliary space. First contribution regards a tighter analysis of the Patience Sorting based approach introduced by Kiyomi et al., showing that the length can be computed in $O\!\left(\max(n \log n, n^2/s)\right)$ time. We also provide an alternative divide-and-conquer method for recovering the sequence, which combined with the improved length analysis shows an algorithm working in $O\!\left(\max(n \log n \log \log s, n^2/s)\right)$ time.

\subsection{New results}
\subsubsection{Length computation}
We provide a tighter analysis of the patience sorting algorithm, building on the work of Kiyomi et al. and related results. In particular, we show that the algorithm runs in $O\!\left(\max(n \log n, n^2/s)\right)$ time, improving on the previously stated $O(n^2 \log n / s)$ bound.
\subsubsection{LIS computation}
To recover an LIS, we present an alternative divide-and-conquer algorithm which, combined with the improved length bound, runs in $O\!\left(\max(n \log n \log \log s, n^2/s)\right)$ time. This improves on the earlier $O(n^2 \log^2(s) / s)$ result.
