\title{\textbf{Longest Increasing Subsequence Problem in Small Space, revisited\\}}

\date{\today}

\begin{document}

\maketitle

\begin{abstract}
Given an input array of comparable elements, we are to find either a longest increasing subsequence, or its length using parametrised space.
It is well known that this problem can be solved in $\mathcal{O}(n \log n)$ time without constrained memory. For parametrised memory - $s$, where
$\sqrt{n} \leq s \leq n$ we present an improved analysis of $Patience\ Sorting$ algorithm presented in \cite{kiyomi2017space}. We show that the length computation runs in $\mathcal{O}(\max(n \log n, \frac{n^2}{s}))$ time.
We also show an improved $D \& C$ algorithm for obtaining the sequence running in $\mathcal{O}(\max(n \log n \log \log s, \frac{n^2}{s}))$ time, while using $\mathcal{O}(s \log n)$ bits.
\end{abstract}

\section{Introduction}
The longest increasing subsequence (LIS for short) problem is a classic algorithmic problem with applications in areas such as sequence analysis, pattern discovery, and optimization. Given a sequence of numbers, the task is to find a longest subsequence whose elements appear in strictly increasing order while preserving their original order, or to compute length of such sequence.
Although the LIS problem has been studied extensively in many variants, the trade-off between time and space remains relevant when the input is large or memory is limited. In this paper, we focus on computing both the length of an LIS and an actual LIS using small auxiliary space. First contribution regards a tighter analysis of the Patience Sorting based approach introduced by Kiyomi et al.~\cite{kiyomi2017space}, showing that the length can be computed in $O\!\left(\max(n \log n, n^2/s)\right)$ time. We also provide an alternative divide-and-conquer method for recovering the sequence, which combined with the improved length analysis shows an algorithm working in $O\!\left(\max(n \log n \log \log s, n^2/s)\right)$ time.

\section{Preliminaries}

We define two variants of the Longest Increasing Subsequence problem.

\problembox
{Longest increasing Subsequence (\Lis)}
{A sequence of $n$ numbers $A = (a_1, a_2, \dots, a_n)$.}
{The longest subsequence $(a_{i_1}, a_{i_2}, \dots, a_{i_k})$ of $A$ such that $i_1 < i_2 < \dots < i_k$ and $a_{i_1} < a_{i_2} < \dots < a_{i_k}$.}

\problembox
{Lenght of \Lis \ (\LisLen)}
{A sequence of $n$ numbers $A = (a_1, a_2, \dots, a_n)$.}
{$k$ - maximal integer such that there exist an increasing subsequence in $A$ of length $k$}


We use the RAM model, where the input sequence is read-only, and the working memory is limited to $\mathcal{O}(s)$ words.

\subsection{Patience Sorting}

Our algorithm is based on the classic \Psort. Let us define basic notions use throughout the algorithm. Idea of the algorithm is based
on maintaining $Piles$\ -- stacks of integers such that for all $x \in A$, $x$ belongs to the $i$-th pile iff. the $longest\ increasing\ subsequence$ of $A$ ending in $x$ has length $i$.
Since \Psort\, both classic, and space-parametrised is based on sequential passes through the input sequence we define
pile operations with respect to time when they are being used.

\begin{definition}
For integers $i, t$ we refer to the state $i$-th pile after reading first $t$ elements of the input sequence as $P_{i}^{t}$, and $P_i$ will represent the $i$-th pile after reading the whole input sequence. \\
Let $P$ be some state of the pile (for example $P_3^4$ or $P_2$).
\begin{itemize}
\item $|P|$ is the number of elements that belongs to $P$
\item $\operatorname{top}(P)$ refers to the element on the top of the pile (the one that was added most recently to it), or null if $P$ is empty.
\item For integer $j$, $P[j]$ is the $j$-th element of $P$, where $\operatorname{top}(P) = P[|P|]$, and $P[0]$ is the element on the bottom of $P$. 
\end{itemize}
\end{definition}

It follows from the definition of pile that if we are solving the \LisLen\ problem, then our goal is to compute the largest integer $k$ such that $P_k$ is non-empty.
Classic \Psort\ algorithm achieves that by constructing all piles during a single passing through the input sequence. When reading the $t$-th element of $A$ - $a_t$, we find
the smallest $i$ such that $\operatorname{top}(P_i^{t-1}) \geq a_t$ and we push $a_t$ to the top of the $i$-th pile, or we create a new, rightmost pile with a single element $a_t$.
Below is depicted the \Psort\ algorithm for solving \LisLen.

\begin{algorithm}
    \caption{\Psort}
    \begin{algorithmic}[1]
        \State initialize $P_{0}$ with the single element $-\infty$
        \State set $k := 0$
        \For{$x \in A$}
            \If{$x > \text{top} (P_{k})$}
                \State set $k := k + 1$
                \State create an empty $Pile$, $P_{k}$, and push $x$ to $P_{k}$
            \Else
                \State set $h$ to be the smallest index with $x \leq \text{top}(P_h)$, and push $x$ to $P_h$
            \EndIf
        \EndFor
        \State \Return $k$
  \end{algorithmic}
\end{algorithm}


\begin{observation}\label{obs:pile}
This procedure correctly computes piles, in other words. \\
Element $x \in P_i$ iff. a $longest\ increasing\ subsequence$ of $A$ ending at $x$ is of length $i$
\end{observation}

\subsection{Space parametrised Patience Sorting}
Assume now that our working memory is limited to $\mathcal{O}(s)$ words, where $\sqrt{n} \leq s \leq n$.
We define the space-paramerised version of \Psort\ that originates from \cite{kiyomi2017space}. 
In contrast to the classic \Psort, in this version of algorithm we will pass through $A$ multiple times, each time preserving a single pile in memory.
We will call a single pass through a \Ppass. This procedure assumes that we have a pile $P_i$ stored in memory and calculates another pile $P_j$ guaranteeing that $i + s < j < i + 2s$.
Single \Ppass is consisted of 2 passes through $A$. 
The first one is used to compute $|P_k|$ for $i < k \leq i + 2s$ to establish $j$, the second one returns $P_j$, or $\operatorname{top}(P_j)$ if $j$ is \LisLen. 

\begin{observation}
Suppose we are reading the $t$-th element of $A$ -- $a_t$ and $a_t \in P_j$. Let $i$ be some fixed integer. \\
If $j \neq i$ and $b = \operatorname{top}(P_i^{t-1})$, then 
\begin{itemize}
\item $a_t < b$ if $j < i$
\item $a_t > b$ if $j > i$
\end{itemize} 
\end{observation}
This observation allows us to define the $filtering$ method for the \Ppass\ procedure. We will process the input like in \Psort, but we will $filter\ out$ (ignore)
the elements from $P_k$ where $k < i$ or $k > i + 2s$. Aside from $P_i$ we maintain an index $h$ of the most recently read element that belongs to $P_i$. 
Reading an element $x$ can lead to three cases:
\begin{itemize}
\item If $x$ is $P_i(h+1)$ (the next element of $P_i$) $\rightarrow h := h+1$
\item Else if $x < P_i(h)$, then $x \in P_k$ where $k < i$ $\rightarrow x$ is ignored
\item Otherwise $x > P_i(h)$, then $x \in P_k$ where $k > i$ 
\end{itemize}
In the last case, we need to check if $k > i + 2s$, and calculate the exact value of $k$ if not. Thus it leads to two additional cases:
\begin{itemize}
\item If no element among those read before $x$ belong to $P_{i+2s}$, then $x \in P_k$, $k \leq i + 2s$
\item Otherwise, let $y := \text{top}(P_{i+2s})$. We compare $x$ and $y$
    \begin{itemize}
        \item If $x > y$, then $x \in P_k$, $k > i + 2s$ $\rightarrow x$ is ignored.
        \item Otherwise $x \in P_k$, $k < i + 2s$. 
    \end{itemize}
\end{itemize}
Notice that in all but two of the cases we have made $\mathcal{O}(1)$ operations while processing element $x$. Let us call what we do in rest of the cases
an $expensive\ filter$. 

\begin{observation}
$expensive\ filter$ takes $\mathcal{O}(\log s)$ time. Notice that $expensive\ filter$ purpose is to find $k$, where $i < k < i+2s$, such that $x \in P_k$.
Then, if we maintain top elements of the $Piles$, using their monotonicity, we can find $k$ via binary searching over interval of length $\mathcal{O}(s)$.
\end{observation}




\begin{algorithm}
    \caption{First pass in \Ppass, assume $P_i$ is given, and $|P_i| < s$}
    \begin{algorithmic}[1]
    \State set $h := 0$ \Comment{\textcolor{gray}{$h$ points to the most recently read element in $P_{i}$}}
    \State set $r := 0$ \Comment{\textcolor{gray}{Is the index of the current rightmost $Pile$}}
    \State set $p_{i+1}, p_{i+2}, \ldots, p_{i+2s}$ to $\infty$ \Comment{\textcolor{gray}{$p_k$ is the current $\text{top}(P_{k})$}}
    \State set $c_{i+1}, c_{i+2}, \ldots, c_{i+2s}$ to $0$ \Comment{\textcolor{gray}{$c_k$ is the current size of $P_k$}}
    \For{$ x \in A$} 

        \If{$x$ is $P_i(h+1)$} \Comment{\textcolor{gray}{$x$ is the next element in $P_i$}}
            \State set $h := h + 1$ 
            \State continue
        \ElsIf {$x < P_i(h)$ or $x > p_{i+2s}$} \Comment{\textcolor{gray}{$x$ belongs to the $P_k$ where $k < i \vee k > i+2s$}}
            \State continue
        \EndIf

        \If{$x > p_r$}
            \State set $r := r + 1$ and $k := r$
        \Else
            \State set $k$ to be the smallest such that $x < p_k$
        \EndIf
            
        \State set $p_k := x$ and $c_k := c_k + 1$
    \EndFor
    
    \State set $j$ to be the biggest, such that $0 < c_j \leq s$
    
    \Return $j$
    \end{algorithmic}
\end{algorithm}

It is easy to see how to modify the procedure to compute the $P_j$ if $j$ is given.

\begin{observation}
Since $s > sqrt{n}$ there is at most $frac{n}{s} \leq s$ $Piles$ with more than $s$ elements which implies that $j > i + s$ 
\end{observation}

\section{New results}

\subsection{Length computation}
We provide a tighter time complexity analysis of the \Psort algorithm presented in \cite{kiyomi2017space}. 
We show that the algorithm runs in $\mathcal{O}(\max(n \log n, \frac{n^2}{s}))$ time, improving the previous $\mathcal{O}(\frac{n^2}{s} \log n)$ bound.

\begin{definition}{name}
Let $j_i$ be the index of the $Pile$ kept in memory after $i$-th \Ppass\ with $j_0=0$
\end{definition}

\begin{definition}{name}
Let $W_i$ be the number of $expensive\ filter$ operations during the $i$-th \Ppass\
\end{definition}

\begin{definition}{name}
Let $K$ be the total number of \Ppass -es
\end{definition}

Notice that time complexity of space parametrised \Psort\ for $\sqrt{n} \leq s \leq n$ can be then written as:
\[
T(n,s) = \mathcal{O}(nK + \log s * \sum_{i=1}^K W_i).
\]

\begin{claim} \label{claim:linear-filters}
The total number of $expensive\ filter$ operations is linear, i.e.,
\[
\sum_{i=1}^K W_i = \mathcal{O}(n).
\]
\end{claim}

\begin{proof}
For each pass $i \in \{1, \dots, K\}$, let us define the active interval $I_i = (j_{i-1}, j_{i-1}+2s]$.

Fix a pass $i$. During the first phase of this pass, the $expensive\ filter$ is invoked only for elements whose pile index belongs to $I_i$. Once the new boundary $j_i$ is determined, the second phase invokes the $expensive\ filter$ only if the pile index is in $(j_{i-1}, j_i]$. By the properties of the algorithm, we know that $j_i \le j_{i-1} + 2s$, which means $(j_{i-1}, j_i] \subseteq I_i$. Thus, we can see that:
\[
W_i \le \sum_{k \in I_i} |P_k| + \sum_{k=j_{i-1}+1}^{j_i} |P_k| \le 2\sum_{k \in I_i} |P_k|.
\]
This gives us: 
\[
\sum_{i=1}^K W_i \le 2\sum_{i=1}^K \sum_{k \in I_i} |P_k|.
\]

By \cite{kiyomi2017space} we have $j_i \ge j_{i-1}+s$. This implies that the left endpoints of the intervals $I_i$ advance by at least $s$ in each pass. Since each interval $I_i$ has a width of exactly $2s$, any pile index $k$ can belong to at most two intervals $I$. Therefore, summing the pile sizes over all intervals yields at most twice the total number of elements:
\[
\sum_{i=1}^K \sum_{k \in I_i} |P_k| \le 2\sum_{k=1}^n |P_k| = 2n.
\]
Thus, we get $\sum_{i=1}^K W_i \le 4n = \mathcal{O}(n)$.
\end{proof}

It is established in \cite{kiyomi2017space} that $K \text{ is } \mathcal{O}(\frac{n}{s})$, which gives us the total running time:
\[
T(n,s) = \mathcal{O}(\frac{n^2}{s}) + \mathcal{O}(n)\log s \text{ which is } \mathcal{O}(max(n\log n, \frac{n^2}{s})).
\]

\subsection{LIS computation}
To \Lis, we present an alternative divide-and-conquer algorithm, which combined with the improved length bound, runs in $\mathcal{O}(\max(n \log n \log \log s, \frac{n^2}{s}))$ time. This improves on the earlier $\mathcal{O}(\frac{n^2}{s} \log^2(s))$ result.

\begin{definition}
For $x \in A$ let $\textsc{Lis}_x$ be the $longest\ increasing\ subsequence$ of $A$ ending in $x$.
\end{definition}

\begin{definition}
For integer $x$, let:
\begin{itemize}
\item $L_{\leq x}$ be a valid answer to \Lis\ for $\{y \in A \mid y \leq x \}$
\item $L_{>x}$ be a valid answer to \Lis\ for $\{y \in A \mid y > x \}$
\end{itemize}
\end{definition}

Let $A_{\ell}$ and $A_r$ be the left and right halves of $A$ respectively. Our goal is to efficiently find an element $m$ such that
concatenation of $L_{\leq m}$ for $A_\ell$ and $L_{> m}$ for $A_r$ is an answer to \Lis for original input $A$.

\begin{definition}
For $x \in A$ let pred$(x) \in A_\ell$ be such that there is a valid $\textsc{Lis}_x$ in $A$
such that the last element of it belonging to $A_\ell$ is pred$(x)$
\end{definition}

\begin{claim} Let $x \in P_i$, then \\
pred$(x) = \begin{cases}
x & \text{if } x \in A_\ell \\
null & \text{if } x \in A_r \land i = 1 \\
\text{pred}(y) & \text{where } y = \text{top}(P_{i-1}) \text{ when } x \text{ was being read }
\end{cases}$
\end{claim}

\begin{proof}
If $x \in A_\ell$ its trivial, so assume $x \in A_r$. Now, if $x \in P_1$, then the $\textsc{Lis}_x$ is of
length $1$, thus it is $(x)$, so the pred$(x)$ does not exist. Now, assume that $x \in A_r \land x \in P_i \land i > 1$. Then  
$\textsc{Lis}_x$ is of length $i$. Since at the moment of reading $x$, $y = \text{top}(P_{i-1})$ is smaller than $x$, and $x$ is later in $A$ than $y$, if we append $x$ to $\textsc{Lis}_y$ it is still a valid $increasing\ subsequence$.
Notice that since length of $\textsc{Lis}_y$ is $i-1$, by appending $x$ to it we obtained $\textsc{Lis}_x$. Since $x \in A_r$ the last element belonging to $A_\ell$ in $\textsc{Lis}_x$ is the same one that in $\textsc{Lis}_y$ making pred$(x)$ = pred$(y)$.  
\end{proof}
